Оформление репозитория. Readme.md
README.md - описание проекта в репозиторие

Оформляется с помощью языка разметки "маркдаун" - .md

1) Заголовки:
# H1
## H2
### H3
#### H4
##### H5
###### H6 

2) Черта под заголовком:
Alt-H1
------ 

3) Перенос: 2 пробела в конце строки

4) Курсив: *звёздочки* или _подчёркивания_

5) Полужирное начертание: **двойные звёздочки** или __двойные нижние подчёркивания__

6) Зачёркивание: ~~Двойные тильды~~

7) Списки нумеруются цифрой и точкой
1.
2.

8) Ненумерованный список делается звёздочкой с пробелом
* элемент1
* элемент2

9) Ссылка: текст заключают в квадратные скобки, рядом адрес в круглых скобках [текст](адрес)

либо [адрес ссылки] - когда не нужно вкладывать в текст

Всплывающая подсказка в ссылке: [тескт](адрес ссылки "подсказка")

10) Код оформляется в тройные косые кавычки, после открывания кода пишется язык прогр:
```javascript
code code code
code code code
```


11) Что писать в README.md?
1. Заголовок с названием проекта.

2. Описание проекта: что он делает, какую задачу решает, какую боль закрывает? Если вы выполняете очередную проектную работу, напишите, в чём её отличие от предыдущей.
Также в описании можно рассказать о технологиях, которые вы применяли в проекте.

3. Инструкция по использованию проекта и системные требования: версия языка, необходимые расширения. Цель такой инструкции — дать возможность другим разработчикам запустить проект у себя.

4. Планы по доработке проекта, если они есть. Например: «исправить X с помощью Y, чтобы получить Z». Если планов по доработке нет, не пишите о планах вовсе. Лучше ничего, чем абстрактное «сделать рефакторинг». В общем, чем конкретнее — тем лучше.
